# 全連人員管理系統 - PWA 版本

## 📱 安裝為 App

### 在手機上安裝
1. **Android（Chrome）**：
   - 開啟 `全連人員管理系統v2.html`
   - 點右上角「⋮」→ 「安裝應用程式」（或「新增到主螢幕」）
   - App 會出現在首頁

2. **iPhone（Safari）**：
   - 開啟 `全連人員管理系統v2.html`
   - 點分享按鈕 → 「加入主螢幕」
   - App 會出現在 App 資料庫

### 在電腦上使用
- 直接在瀏覽器開啟 `全連人員管理系統v2.html`
- 或在 Chrome 地址欄右側點「安裝」按鈕

### 本地開發環境說明
- ⚠️ Service Worker 需要 **HTTPS** 或 **localhost** 才能正常運作
- 💡 如果你是直接用檔案瀏覽器開啟（file://），Service Worker 不會註冊，但**系統仍可正常使用**，只是沒有離線功能
- ✅ 資料保存功能在任何情況下都能正常運作，與 Service Worker 無關

---

## 🔄 資料保留 - 自動同步機制

**重要：資料 100% 保留，無需備份**

### 運作原理
- ✅ 所有資料存在瀏覽器本地（localStorage）
- ✅ 每次更新 code，Service Worker 自動檢查新版本
- ✅ 新版本到時自動快取，**原有資料完全保留**
- ✅ 系統自動重新加載，資料無縫接續

### 手動檢查更新
- App 會每小時自動檢查一次更新
- 有更新時會跳出「系統已更新」提示並重載
- 或手動重啟 App 也會檢查

### 手動匯出備份
- 在系統內點「📤 匯出」下載 JSON 檔案
- 可保存在電腦或雲端做額外備份

### 手動匯入還原
- 如需還原備份：點「📥 匯入」選擇 JSON 檔案
- 系統會用該檔案覆蓋當前資料

## ⚙️ 技術細節

### Service Worker 快取策略
- **HTML 檔案**：網路優先（新版本優先，無網路時用快取）
- **字體、外部資源**：快取優先（快速載入，自動更新）
- **其他請求**：直接通過

### 版本控制
- Service Worker 版本 = `v4.1`（每次 code 更新會遞增）
- 舊版快取自動清除，確保不佔空間
- 資料始終保在 localStorage，不受快取影響

## 🛠️ 常見問題

**Q: 為什麼看到「Service Worker 註冊失敗」？**
A: 這在本地開發環境很正常。Service Worker 需要 HTTPS 或 localhost。但不用擔心，**資料保存功能完全不受影響**，系統照常可用。

**Q: 沒有 Service Worker 會怎樣？**
A: 
- ❌ 無法完全離線使用（需要載入外部字體時會失敗）
- ✅ 但資料保存、修改、匯出都正常
- ✅ 上傳到支援 HTTPS 的伺服器後就會自動註冊

**Q: 更新後資料會不會遺失？**
A: 不會。資料存在 localStorage，與 Service Worker 快取完全分開。永遠不會遺失。

**Q: 沒有網路能用嗎？**
A: 大部分功能可以。字體載入需要網路，但已快取過的話離線也能用。

**Q: 如何確認已安裝成 App？**
A: 打開時網址欄會消失，頂部只有系統狀態列。

**Q: 可以同時在多台裝置用嗎？**
A: 可以，但資料各自獨立存儲。如需同步可用「匯出/匯入」功能。

**Q: 如何卸載？**
A: 和卸載一般 App 一樣。資料會被清除，除非你先「匯出」了備份。
